# Bước tiếp theo

Bạn đã hoàn thành việc thiết lập miniapp webview trên local. Dưới đây là các bước tiếp theo.

## Deploy lên Tammi Platform

Để miniapp có thể hoạt động với đầy đủ tính năng (SSO, Bridge API), bạn cần deploy lên Tammi Platform.

| Bước                     | Mô tả                                           |
| ------------------------ | ----------------------------------------------- |
| **1. Đăng ký tài khoản** | Liên hệ Viettel để được cấp tài khoản developer |
| **2. Tạo miniapp**       | Đăng ký thông tin miniapp trên Platform         |
| **3. Build & Upload**    | Build production bundle và upload lên Platform  |
| **4. Submit review**     | Gửi miniapp để được xét duyệt                   |
| **5. Publish**           | Phát hành miniapp cho người dùng                |

:::info Tài liệu deploy
Hướng dẫn chi tiết quy trình deploy sẽ được cung cấp sau khi bạn có tài khoản developer.
:::

## Giới hạn của phương thức Webview

Khi sử dụng phương thức nhúng webview, bạn cần lưu ý các giới hạn sau:

| Giới hạn                | Giải thích                                                                                        |
| ----------------------- | ------------------------------------------------------------------------------------------------- |
| **Bridge API hạn chế**  | Chỉ có thể gọi Bridge API tại thời điểm khởi tạo miniapp, không thể gọi trong runtime của webview |
| **Không có hot reload** | Webview load từ URL cố định, không có live reload như phát triển SPA                              |
| **Phụ thuộc network**   | Website phải accessible từ thiết bị mobile                                                        |

:::tip Khi nào nên chuyển sang phương thức khác?
Nếu ứng dụng của bạn cần sử dụng **nhiều Bridge API** hoặc cần **tương tác sâu** với native features, hãy cân nhắc [phương thức xây mới](../2_quick_start/) hoặc [chuyển đổi SPA](../1_gioi_thieu/lua_chon_tich_hop).
:::
